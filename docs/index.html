Követelmények, védések

Minimális követelmény

Az aláírás feltétele (a legfeljebb három hiányzáson felül) legalább egy-egy házi feladat elkészítése mindkét programvédés anyagából.

Védések, pótvédések

A gyakorlaton megszerzett tudás számonkérése programvédések formájában történik. A félév során két programvédésre kerül sor: az első védésen az 1-2-3. házi feladatok, a második védésen a 4-5. házi feladatok bemutatása lehetséges.

A normál védésen nem elfogadott vagy nem bemutatott programok megvédése az úgynevezett pótvédéseken lehetséges.

Programvédésen olyan forráskód mutatható be, melyet a hallgató feltöltött a megfelelő házi feladathoz számára létrehozott GitHub repositoryba!

A védések menete

A hallgató elkészíti a házi feladatot, azaz a kódot rendszeresen feltölti a számára az adott feladathoz létrehozott GitHub repositoryba.

A gyakorlatvezető a védést megelőzően, az *utolsó effektív commit* időpontja előtti utolsó feltöltött commitnak megfelelő állapotban megvizsgálja a repository tartalmát. Ez magában foglalja a csalások kiszűrését is.

A védésen a hallgató megmutatja futás közben a programot, majd pedig a gyakorlatvezető kérdéseire adott válaszokon keresztül elmagyarázza a program működését.

Felhasználható eszközök

A házi feladatok implementálásához kizárólag szabványos C/C++ használható (akár C++20 is). Ez azt jelenti, hogy a házi feladatokat platform- és fordítófüggetlen módon kell elkészíteni.

A C/C++ standard könyvtáron felül felhasználhatók a következők:

  • a GLUT vagy GLFW ablakozókönyvtár,
  • a Bevezetés a számítógépi grafikába tárgyhoz kiadott matematikai fejlécállomány,
  • a NanoGUI GUI könyvtár.
